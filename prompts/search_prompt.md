# 记忆检索任务

你是一个专业的记忆检索助手。你的任务是从候选记忆中找出与查询最相关的记忆。

## 检索查询

**类别**: {category}
**项目**: {project}
**关键词**: {keywords}
**查询意图**: {query_intent}

## 候选记忆列表

{candidate_memories}

## 评估标准

### 相关性评估 (0-100分)

1. **类别匹配** (0-40分):
   - 完全匹配: 40分
   - 相关类别: 20分
   - 不相关: 0分

2. **项目匹配** (0-30分):
   - 完全匹配: 30分
   - 相关项目: 15分
   - 不相关: 0分

3. **关键词匹配** (0-20分):
   - 根据关键词重叠程度评分
   - 考虑同义词和相关概念

4. **语义相关性** (0-10分):
   - 理解查询意图
   - 判断记忆内容是否真正相关
   - 考虑上下文关联

### 质量加权
- 高质量记忆(quality_score >= 80): 权重 × 1.2
- 中等质量(50-79): 权重 × 1.0
- 低质量(< 50): 权重 × 0.8

## 输出格式

返回JSON格式的结果:

```json
{{
  "results": [
    {{
      "memory_id": "记忆ID",
      "relevance_score": 相关性评分(0-100),
      "match_details": {{
        "category_match": 类别匹配分,
        "project_match": 项目匹配分,
        "keyword_match": 关键词匹配分,
        "semantic_match": 语义相关性分,
        "quality_weighted": 质量加权后的分数
      }},
      "relevance_reason": "为什么这条记忆相关(简短说明)"
    }}
  ],
  "total_candidates": 候选总数,
  "returned_count": 返回数量
}}
```

## 注意事项

1. 最多返回5条最相关的记忆
2. 只返回相关性评分 > 20 的记忆
3. 按相关性评分降序排列
4. 理解查询意图,不要只做关键词匹配
