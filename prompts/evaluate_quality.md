## 评估维度

### 1. 复用性 (权重: 40%)

**评分标准**:
- **高 (80-100分)**: 
  - 通用解决方案,可应用于多个项目
  - 可复用的模式、算法、架构
  - 最佳实践或经验总结
  
- **中 (50-79分)**:
  - 特定场景的解决方案
  - 项目相关的经验
  - 有一定复用价值
  
- **低 (0-49分)**:
  - 一次性操作
  - 临时解决方案
  - 项目特定配置

**评估要点**:
- 这个解决方案能否用于其他项目?
- 是否包含可复用的模式或方法?
- 是否是通用最佳实践?

### 2. 复杂度 (权重: 30%)

**评分标准**:
- **高 (80-100分)**:
  - 复杂架构设计
  - 高难度算法实现
  - 多系统集成方案
  
- **中 (50-79分)**:
  - 中等复杂度的实现
  - 需要一定思考的方案
  - 多步骤操作
  
- **低 (0-49分)**:
  - 简单操作
  - 常规配置
  - 基础实现

**评估要点**:
- 解决方案的复杂程度如何?
- 是否需要深入理解才能实现?
- 是否包含技术难点?

### 3. 独特性 (权重: 20%)

**评分标准**:
- **高 (80-100分)**:
  - 创新解决方案
  - 独特见解或发现
  - 非常规方法
  
- **中 (50-79分)**:
  - 改进的常规方案
  - 有一定优化
  - 结合多种方法
  
- **低 (0-49分)**:
  - 常规方案
  - 标准实现
  - 无特殊之处

**评估要点**:
- 这个方案是否独特?
- 是否有创新点?
- 是否比常规方案更好?

### 4. 时效性 (权重: 10%)

**评分标准**:
- **高 (80-100分)**:
  - 长期有效(数年)
  - 基础原理或模式
  - 不易过时
  
- **中 (50-79分)**:
  - 中期有效(数月)
  - 当前技术栈相关
  - 可能随版本变化
  
- **低 (0-49分)**:
  - 短期有效(数周)
  - 临时方案
  - 易过时

**评估要点**:
- 这个记忆能保持多长时间的有效性?
- 是否会随技术发展而过时?
- 是否是临时解决方案?

## 计算公式

```
quality_score = 复用性 × 0.4 + 复杂度 × 0.3 + 独特性 × 0.2 + 时效性 × 0.1
```

## 决策规则

- **quality_score >= 70**: 创建完整记忆
- **50 <= quality_score < 70**: 创建压缩记忆
- **quality_score < 50**: 不创建记忆

## 注意事项

1. 客观评分,不要过度评分
2. 考虑实际复用价值,而非理论价值
3. 独特性评分要谨慎,大多数方案都是常规方案
4. 时效性要考虑技术发展趋势
